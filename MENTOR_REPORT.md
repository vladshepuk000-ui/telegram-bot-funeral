## Що було реалізовано

### Етап 1 — Telegram бот
- Реєстрація клієнта (ім'я, телефон)
- Каталог товарів по категоріях (рідини, картриджі, системи)
- Оформлення замовлення через бота
- Вибір доставки: Нова Пошта або самовивіз
- Статуси замовлення: нове → підтверджено → відправлено → виконано / скасовано
- Повторне замовлення з кошику
- Відгуки після доставки (зірочки + текст)
- Сповіщення адміну в Telegram про кожне замовлення

### Етап 2 — Розсилки та планувальник (APScheduler)
- Щопонеділка о 10:00 — розсилка акцій всім клієнтам
- Нагадування клієнтам через 21 день після замовлення
- Щотижневий звіт адміну (замовлення, виручка, нові клієнти)
- Щоденний звіт о 23:00 по статистиці сайту (відвідування, кліки, замовлення)

### Етап 3 — AI-асистент (Groq API)
- Клієнт може запитати AI про товари прямо в боті
- Використано модель llama3 через Groq (безкоштовний API)

### Етап 4 — Веб адмін-панель (FastAPI)
- Авторизація по логіну/паролю з сесією в cookie
- Дашборд: кількість замовлень, виручка, нових клієнтів, активних товарів
- Сторінка замовлень з фільтрами по статусу, зміна статусу
- Сторінка товарів: додати, редагувати, сховати/показати, поповнити залишок
- Позначки "Новинка" та "Хіт продажів" для товарів
- Стара ціна (ефект знижки) — перекреслена ціна
- Сторінка клієнтів
- Сторінка розсилок — надіслати вручну або за розкладом
- Мобільна адаптація панелі (бургер-меню)

### Етап 5 — Публічний сайт (/site)
- Головна сторінка з hero-секцією (анімація парів, фіолетовий градієнт)
- Каталог товарів з фото (через Telegram file proxy)
- Модальне вікно товару з детальним описом і кількома фото
- Кошик (localStorage) — додавання, видалення, зміна кількості
- Форма замовлення прямо на сайті (без Telegram): ім'я, телефон, доставка, місто, Telegram, коментар
- Після замовлення — сповіщення адміну в Telegram і зменшення залишку в базі
- Кнопка скасування замовлення з таймером на 5 хвилин
- Відгуки клієнтів (з бази даних бота)
- Лічильник онлайн-відвідувачів
- Кнопка "Копіювати телефон"
- Хеш-навігація (#product-id) — посилання на конкретний товар
- Автоматичне відображення "немає в наявності" якщо stock = 0

### Етап 6 — Публічний доступ (ngrok)
- Встановлено ngrok для тимчасового публічного посилання
- Сайт доступний за адресою `https://hypnotist-fossil-ferocity.ngrok-free.dev/site`

## Складності та як вирішували

### 1. `&&` не працює в PowerShell
**Проблема:** Команди типу `cd vape_shop && python run_web.py` викидали помилку в PowerShell.  
**Рішення:** Використовувати крапку з комою: `cd vape_shop; python run_web.py`, або запускати команди окремо.

### 2. Таблиця `web_orders` не існувала
**Проблема:** При першому замовленні через сайт — помилка 500, бо таблиця не була створена.  
**Рішення:** Додати `CREATE TABLE IF NOT EXISTS web_orders` в `app.py` при старті застосунку.

### 3. Кнопка "Скасувати замовлення" не з'являлась
**Проблема:** Після успішного замовлення кошик очищувався, `renderCart()` ховав footer — і кнопка скасування зникала разом з ним.  
**Рішення:** Додати перевірку: якщо показується success-стан — не ховати footer.

### 4. Можна було оформити порожній кошик
**Проблема:** Форма замовлення не перевіряла чи є товари в кошику.  
**Рішення:** Додати валідацію на фронтенді перед відправкою.

### 5. Фото товарів — різні розміри на мобільному
**Проблема:** Фото в каталозі мали різні пропорції, картки виглядали нерівно.  
**Рішення:** CSS `object-fit: cover` + фіксована висота контейнера.

### 6. `repeat_order` падало з помилкою
**Проблема:** При повторному замовленні через бота — помилка через неправильну роботу з кошиком.  
**Рішення:** Виправлено логіку повторного замовлення.

### 7. Старий `photo_id` повертав помилку 504
**Проблема:** Telegram API повертав помилку для старих file_id, запит зависав.  
**Рішення:** Додано `aiohttp.ClientTimeout(total=10)` і `try/except` з поверненням 504.

### 8. `request.client` міг бути None
**Проблема:** На деяких середовищах `request.client` — None, що спричиняло краш.  
**Рішення:** Перевірка `ip = request.client.host if request.client else "невідомо"`.

### 9. ngrok запущений двічі
**Проблема:** На безкоштовному плані не можна мати два активні тунелі — сайт не відкривався.  
**Рішення:** Зупинити обидва процеси і запустити один раз.

### 10. Неправильне розширення посилання ngrok
**Проблема:** Сайт не відкривався бо в адресному рядку було `.app` замість `.dev`.  
**Рішення:** Скопіювати посилання прямо з терміналу ngrok.
