# Навчальний проект: Автоматизація вейп-магазину

## Про проект

Я розробляю систему автоматизації для невеликого онлайн-магазину рідин та картриджів для вейпів.
У мене вже є Telegram-канал де я публікую асортимент, але все інше — вручну: відповіді клієнтам, оновлення наявності, пошук нових покупців.

Мета проекту — автоматизувати максимум рутини і при цьому навчитись будувати складні реальні системи.

---

## Чому саме цей проект

- Реальна задача, не вигадана — є конкретні проблеми які потрібно вирішити
- Малий асортимент — фокус на архітектурі системи, а не на складній бізнес-логіці
- Повністю безкоштовний стек — можна запустити в production без жодних витрат
- Охоплює багато технологій одночасно: бекенд, бази даних, API, фронтенд, автоматизація

---

## Які навички отримаю

| Технологія | Що вивчу |
|---|---|
| `aiogram 3.x` | Побудова складних Telegram-ботів з FSM |
| `SQLite / PostgreSQL` | Проектування БД, SQL запити, зв'язки між таблицями |
| `APScheduler` | Планування фонових задач, асинхронність |
| `Telethon` | Робота з Telegram User API, парсинг каналів |
| `FastAPI` | REST API, авторизація, сесії |
| `Vue.js` | Фронтенд, PWA, робота з API |
| `Google Gemini API` | Інтеграція AI у реальний продукт |
| `Railway / Vercel` | Деплой і хостинг застосунків |

---

## Технічний стек

**Бекенд:** Python 3.14, aiogram 3.27, FastAPI, APScheduler, Telethon
**База даних:** SQLite (розробка) → PostgreSQL (production)
**AI:** Google Gemini API (безкоштовний tier)
**Фронтенд:** Vue.js + PWA
**Хостинг:** Railway (бот + API), Vercel (PWA)
**Вартість:** 0 грн/місяць

## Етапи розробки

### Етап 1 — Telegram-бот з каталогом і замовленням
Бот приймає замовлення, відповідає на питання, сповіщає адміна.
Клієнт сам обирає товар і оформлює замовлення без моєї участі.

### Етап 2 — Розсилки і планувальник задач
Авторозсилки по базі клієнтів, нагадування, щотижневі звіти,
автоматичний бекап бази даних.

### Етап 3 — Автопостинг у групи регіону
Бот сам публікує оголошення в тематичні Telegram-групи
сусідніх міст і сіл за розкладом.

### Етап 4 — Парсер Telegram-каналу
Публікую пост у свій канал → бот розпізнає товар і ціну →
каталог в боті оновлюється автоматично.

### Етап 5 — Веб-дашборд
Адмін-панель у браузері: замовлення, товари, статистика продажів.
REST API для фронтенду.

### Етап 6 — AI-асистент
Клієнт пише будь-яке питання — Gemini AI відповідає замість мене,
з урахуванням інформації про магазин і контексту розмови.

### Етап 7 — Веб-сайт магазину
Повноцінний сайт магазину на Vue.js: каталог, корзина, оформлення замовлення,
перевірка статусу. Адаптивний дизайн — зручно на телефоні і комп'ютері.
