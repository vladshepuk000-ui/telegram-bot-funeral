# Покроковий план дій

## Правила роботи
- Один крок = одна задача, не більше
- Не переходь далі поки поточний крок не протестований
- Після кожного кроку — git commit з описом що зробив

---

# ЕТАП 1 — Бот з каталогом і замовленням

### Крок 1 — Підготовка середовища
1. Відкрий `vape_shop/` у VSCode
2. Створи папки: `bot/handlers/`, `bot/keyboards/`, `bot/states/`, `database/`, `logs/`
3. Скопіюй `.env.example` → `.env`, заповни `BOT_TOKEN` і `ADMIN_IDS`
4. Встанови залежності: `py -m pip install -r requirements.txt`
5. Перевір що бот токен правильний: запусти `py -c "from aiogram import Bot"`

---

### Крок 2 — База даних
1. Створи `database/init_db.py` з функцією `create_tables()`
2. Опиши таблиці: `products`, `customers`, `orders`, `order_items`
3. Запусти `py database/init_db.py` → має з'явитись `vape_shop.db`
4. Створи `database/queries.py` — базові функції читання і запису

---

### Крок 3 — Запуск і /start
1. Створи `bot/main.py` — Dispatcher, Router, запуск polling
2. Створи `bot/handlers/start.py` — обробник /start
3. /start: зберігає клієнта в БД, показує меню з 4 кнопками
4. Запусти: `py bot/main.py` → протестуй /start у Telegram

---

### Крок 4 — Каталог
1. `bot/keyboards/catalog_kb.py` — кнопки категорій і товарів
2. `bot/handlers/catalog.py` — /catalog показує категорії
3. Клік по категорії → список товарів з фото і ціною
4. Клік по товару → картка: фото, опис, ціна, кнопка "Замовити"
5. Додай 2-3 тестових товари вручну в БД
6. Протестуй весь каталог

---

### Крок 5 — FAQ
1. `bot/handlers/faq.py` — список питань кнопками
2. При натисканні — відповідь з тексту в `docs/faq.md`
3. Протестуй всі питання

---

### Крок 6 — FSM замовлення
1. `bot/states/order_states.py` — StatesGroup:
   `choosing_product → choosing_quantity → entering_address → entering_phone → entering_notes → confirmation`
2. `bot/handlers/order.py` — обробник кожного кроку
3. Валідація телефону через regex на кроці `entering_phone`
4. Крок `confirmation` — підсумок замовлення + реквізити оплати
5. /cancel скидає стан в будь-який момент
6. При підтвердженні — зберегти в БД
7. Протестуй повний флоу: від /start до підтвердження

---

### Крок 7 — Сповіщення
1. Після замовлення — адмін отримує всі деталі
2. Клієнт отримує підтвердження + що робити далі (оплата)
3. Протестуй: зроби тестове замовлення, перевір обидва повідомлення

---

### Крок 8 — Адмін-команди
1. `bot/handlers/admin.py` — перевірка `user_id in ADMIN_IDS` на кожній команді
2. /addproduct — FSM: назва → категорія → опис → ціна → фото → кількість
3. /removeproduct — вибір зі списку → підтвердження
4. /orders — останні 10 замовлень зі статусом
5. /setstatus `[id] [статус]` — змінити статус + повідомити клієнта
6. Протестуй кожну команду

---

# ЕТАП 2 — Розсилки + планувальник

### Крок 9 — APScheduler
1. В `bot/main.py` підключи APScheduler — запускається разом з ботом
2. Створи `scheduler/tasks.py`:
   - `check_stock()` — щодня 09:00
   - `weekly_report()` — щопонеділка 08:00
   - `backup_db()` — щодня 03:00, надсилає файл БД адміну
3. Зареєструй задачі при старті
4. Протестуй: викликай `check_stock()` вручну через код

---

### Крок 10 — Розсилки
1. /broadcast для адміна — FSM: текст → фото (опційно) → підтвердження → розсилка всій базі
2. APScheduler: щопонеділка о 10:00 авторозсилка заготовленого тексту
3. Нагадування клієнту через 21 день після замовлення
4. Протестуй /broadcast на собі

---

# ЕТАП 3 — Автопостинг у групи

### Крок 11 — Авторизація Telethon
1. Отримай API_ID і API_HASH на my.telegram.org
2. Створи `autoposter/poster.py`
3. Перша авторизація: введи номер і код — зберігається `.session` файл
4. Надішли тестове повідомлення самому собі

---

### Крок 12 — Шаблони і розклад
1. Створи 5-7 шаблонів оголошень у конфіг-файл
2. Збери список груп регіону (вручну)
3. APScheduler: пост 2 рази на день, кожен раз інший шаблон
4. Затримки між групами 30-60 секунд (захист від бану)
5. Логування в `logs/autoposter.log`

---

# ЕТАП 4 — Парсер каналу

### Крок 13 — Слухач і парсер
1. `channel_parser/parser.py` — Telethon слухає нові пости в каналі
2. `channel_parser/recognizer.py` — функції розпізнавання:
   - `extract_price(text)` — regex для ціни (число + грн/₴)
   - `extract_category(text)` — ключові слова: рідина/картридж/система
   - `extract_name(text)` — перший рядок або після ключового слова
3. Якщо товар новий — INSERT в БД; якщо є — UPDATE ціни і наявності
4. Збереження photo_id (для бота) і photo_url (для PWA)
5. Протестуй: напиши тестовий пост у канал, перевір БД

---

# ЕТАП 5 — Веб-дашборд

### Крок 14 — FastAPI бекенд
1. `dashboard/app.py` — FastAPI, авторизація через сесійні куки
2. REST API ендпоінти (потрібні для PWA):
   - `GET /api/products`
   - `GET /api/orders`
   - `GET /api/stats`
   - `POST /api/orders`
3. Запусти: `uvicorn dashboard.app:app --reload`
4. Перевір ендпоінти через браузер або Postman

---

### Крок 15 — Адмін-панель (HTML)
1. `dashboard/templates/index.html` — Bootstrap таблиці
2. Сторінки: замовлення / товари / статистика
3. Деплой на Railway — пов'яжи з GitHub репозиторієм

---

# ЕТАП 6 — AI-асистент (Gemini, безкоштовно)

### Крок 16 — Підключення Gemini
1. Зареєструйся на aistudio.google.com → Get API key (безкоштовно)
2. `ai/prompts.py` — системний промпт з інфо про магазин
3. `ai/assistant.py` — функція запиту до Gemini з контекстом розмови
4. Таблиця `ai_chat_history` в БД — зберігає останні 10 повідомлень клієнта
5. `bot/handlers/ai_chat.py` — інтеграція з ботом
6. Якщо клієнт питає щось чого AI не знає → тег "потрібна допомога" адміну
7. Ліміт: AI_DAILY_LIMIT запитів на клієнта на день

---

# ЕТАП 7 — Веб-сайт магазину

### Крок 17 — Vue.js + деплой
1. `cd website && npm create vue@latest`
2. Встанови axios: `npm install axios`
3. Сторінки: головна, каталог, картка товару, корзина, оформлення, статус замовлення
4. Підключи до REST API (Етап 5)
5. Адаптивний дизайн — Bootstrap або Tailwind CSS
6. Деплой: `npx vercel` (безкоштовно)
