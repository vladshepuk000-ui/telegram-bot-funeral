# Roadmap — Етапи розробки

## Етап 1 — Telegram-бот з каталогом і замовленням
**Мета:** базовий бот який замінює ручні переписки з клієнтами

### Задачі:
- [ ] Структура папок, .env, requirements.txt, .gitignore
- [ ] Ініціалізація SQLite бази даних
- [ ] Таблиці: `products`, `customers`, `orders`, `order_items`
- [ ] /start — привітання, головне меню, збереження клієнта в БД
- [ ] /catalog — каталог з фото, категорії: рідини / картриджі / системи
- [ ] FSM замовлення: товар → кількість → адреса → телефон → коментар → підтвердження
- [ ] /cancel — скидає FSM у будь-який момент
- [ ] /stop — клієнт відписується від розсилок (поле `is_subscribed` в таблиці customers)
- [ ] Сповіщення адміну після нового замовлення
- [ ] Підтвердження клієнту + реквізити для оплати
- [ ] /faq — автовідповіді на часті питання
- [ ] Адмін: /addproduct, /removeproduct, /orders, /setstatus
- [ ] Кнопка "Повторити останнє замовлення" — підтвердження і готово, без FSM заново
- [ ] Кнопка "Сповісти коли з'явиться" — якщо товару немає в наявності
- [ ] Таблиця `waitlist` — хто чекає на який товар
- [ ] При поновленні залишку товару — автосповіщення всім з waitlist

**Результат:** клієнт сам переглядає товари і оформлює замовлення без твоєї участі

## Етап 2 — Розсилки + планувальник задач
**Мета:** система сама нагадує клієнтам і слідкує за магазином

### Задачі:
- [ ] APScheduler підключений до бота (запускається разом)
- [ ] Таблиця `broadcasts` — логування розсилок (кому, коли, текст, скільки отримали)
- [ ] Таблиця `broadcast_templates` — шаблони текстів для авторозсилок (заповнюєш вручну)
- [ ] /broadcast — адмін надсилає повідомлення всім підписаним клієнтам
- [ ] Розсилка надсилає тільки клієнтам з `is_subscribed = true`
- [ ] Якщо клієнт заблокував бота — помилка ігнорується, продовжує далі
- [ ] Авторозсилка щопонеділка о 10:00 — бере черговий шаблон з `broadcast_templates`
- [ ] Нагадування клієнту через 21 день після замовлення
- [ ] APScheduler: через 2 дні після статусу "доставлено" → запит відгуку клієнту
- [ ] Таблиця `reviews` — зберігає оцінку (1-5) і текст відгуку
- [ ] Адмін бачить нові відгуки в дашборді
- [ ] Сповіщення адміну якщо товар на нулі (щодня о 09:00)
- [ ] Щотижневий звіт адміну: скільки замовлень, яка виручка
- [ ] Авторезервна копія БД щодня о 03:00 → тобі в Telegram файлом

**Результат:** клієнти повертаються самі, ти отримуєш тільки важливі сповіщення

---

## Етап 3 — Автопостинг у групи регіону
**Мета:** нові клієнти з сусідніх сіл і міст без ручної роботи

### Задачі:
- [ ] Авторизація Telethon (один раз, зберігається сесія)
- [ ] Зібрати список груп регіону вручну → зберегти в `autoposter/groups.txt`
- [ ] 5-7 шаблонів тексту оголошення в `autoposter/templates.txt` з авторотацією
- [ ] Розклад: пости о 12:00 і о 19:00 щодня
- [ ] Захист від бану: пауза 30-60 сек між групами, різний текст щоразу
- [ ] Якщо група заблокувала — пропускає і логує помилку
- [ ] Логування: коли, куди, який текст опубліковано → `logs/autoposter.log`

**Результат:** про тебе дізнаються нові люди без жодних ручних дій

---

## Етап 4 — Парсер твого Telegram-каналу
**Мета:** написав пост у канал — каталог в боті оновився сам

### Задачі:
- [ ] Telethon слухає нові пости в каналі (той самий пакет що і в Етапі 3)
- [ ] Розпізнає з тексту: назву товару, ціну, категорію
- [ ] Фото з поста — зберігає як `photo_id` (для бота) і завантажує на imgbb.com як `photo_url` (для сайту, безкоштовно)
- [ ] Новий товар — додає в БД; існуючий — оновлює ціну/наявність
- [ ] Адмін отримує підтвердження що каталог оновлено

**Результат:** один пост у канал = оновлений каталог у боті і на сайті автоматично

---

## Етап 5 — Веб-дашборд
**Мета:** керувати магазином і бачити статистику через браузер

### Задачі:
- [ ] FastAPI додаток з авторизацією (логін/пароль)
- [ ] REST API для веб-сайту: /api/products, /api/orders, /api/stats
- [ ] Головна: кількість замовлень, виручка, нові клієнти сьогодні
- [ ] Сторінка товарів: додати / змінити / приховати
- [ ] Сторінка замовлень: таблиця зі статусами і фільтрами
- [ ] Деплой на Railway або Render (безкоштовно)

**Результат:** повноцінна адмін-панель доступна з будь-якого пристрою

---

## Етап 6 — AI-асистент на Gemini (безкоштовно)
**Мета:** клієнт пише будь-яке питання — отримує розумну відповідь

### Задачі:
- [ ] Отримати безкоштовний Gemini API ключ на aistudio.google.com
- [ ] Таблиця `ai_chat_history` — зберігає контекст розмови (останні 10 повідомлень)
- [ ] Таблиця `ai_usage` — лічильник запитів на клієнта за поточний день
- [ ] Системний промпт: інфо про магазин, асортимент, умови доставки
- [ ] AI відповідає на вільні питання клієнтів
- [ ] Якщо не знає відповіді → передає адміну з позначкою
- [ ] Ліміт 30 запитів на клієнта на день (налаштовується в .env)
- [ ] /updateprompt — адмін оновлює знання AI без зміни коду

**Результат:** клієнти отримують відповіді 24/7, ти нічого не робиш вручну

---

## Етап 7 — Веб-сайт магазину
**Мета:** клієнт заходить на сайт, переглядає товари і оформлює замовлення

### Задачі:
- [ ] Vue.js проект у папці `website/`
- [ ] Сторінки: головна, каталог, картка товару, корзина, оформлення замовлення
- [ ] Сторінка статусу замовлення — клієнт вводить номер і бачить статус
- [ ] Підключення до REST API з Етапу 5
- [ ] Адаптивний дизайн — зручно на телефоні і комп'ютері
- [ ] Деплой на Vercel (безкоштовно)

**Результат:** повноцінний сайт магазину, працює на будь-якому пристрої без встановлення
